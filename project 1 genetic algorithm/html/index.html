This assignment used a genetic algorithm, search within a very large state-space to find a simple approximation to a given digital image. Following Roger Johansson's idea (http://rogeralsing.com/2008/12/07/genetic-programming-evolution-of-mona-lisa/)
Each image uses P polygons, where P=100 for the results on this page. N is the population size and K is the number of new children created per generation. T is the number of generations displayed in each animation.
The fitness of the image is a measure of how closely it matches the original image.
The learning curve shows how the fitness of the best approximation improves over time.
The 3D graphs depict how the fitness after 25000 generations varies with N and K.

Mutation question

Answer:
With 1/7 probability, modify a polygon by adding or deleting a vertex (with same probability).
With 5/21 probability, modify a vertex by moving it with deviation of 0.05 * image width or height correspondingly.
With 1/7 probability, modify color of a polygon with deviation of 0.1 * 255.
With 1/7 probability, modify alpha of a polygon with deviation of 0.1
With 2/21 probability, swap two random polygons
With 5/21 probability, generate a random polygon. The polygon is generated by first selecting a center point randomly, then generate three vertices with deviation of 0.05 * image width or height.

Parent selection question

Answer:Parents are selected randomly. The probability for each one to be selected is proportional to the square of fitness. This is done by add up all fitness square of all parents, and generate a double number in range (0, sum), then determine which range the number falls in.
